# almaPLUS SuperApp — Видеогид (QR-инструкция)

Статичный сайт: клиент сканирует QR → выбирает язык (KZ/RU) → выбирает тему → смотрит видео.
Никакого сервера и базы данных не нужно — всё работает прямо с GitHub Pages.

## Структура проекта

```
almaplus-guide/
├── index.html              ← весь сайт (один файл: вёрстка + логика)
├── videos/
│   ├── ru/
│   │   ├── 01-tema1.mp4
│   │   ├── 02-tema2.mp4
│   │   └── 03-tema3.mp4
│   └── kz/
│       ├── 01-tema1.mp4
│       ├── 02-tema2.mp4
│       └── 03-tema3.mp4
└── README.md
```

## Схема именования видеофайлов

Используй такой шаблон:

```
<номер>-<короткий-слаг-темы>.mp4
```

Примеры:
- `01-registratsiya.mp4`
- `02-balans.mp4`
- `03-podderzhka.mp4`

Правила:
- только латиница, цифры и дефисы, без пробелов и кириллицы (иначе ссылки могут ломаться на некоторых телефонах);
- номер в начале — чтобы файлы не путались между собой и сортировались по порядку;
- одинаковый номер и тема в обеих папках (ru и kz), просто файл будет на нужном языке внутри.

Когда пришлёшь мне реальные названия тем — просто пришли их списком, и я обновлю и список тем в коде (`CONFIG`), и подскажу финальные имена файлов под каждую тему.

## Как подключить свои 6 видео

1. Положи видеофайлы в папки `videos/ru/` и `videos/kz/` с именами по схеме выше.
2. Открой `index.html`, найди блок `CONFIG` в самом низу файла (внутри `<script>`).
3. Для каждого видео впиши:
   - `file` — путь к файлу, например `"videos/ru/01-registratsiya.mp4"`
   - `title` — название темы, которое увидит клиент, например `"Регистрация в приложении"`

Больше никаких правок делать не нужно — список и плеер подстроятся автоматически.

## Публикация на GitHub Pages

1. Создай новый репозиторий на GitHub (например `almaplus-guide`).
2. Загрузи туда `index.html`, `README.md` и папку `videos/` со всеми файлами.
   - Через веб-интерфейс GitHub: кнопка **Add file → Upload files** (можно перетащить сразу всю папку videos с подпапками, если зайти через "Add file" в нужную вложенную папку, либо проще — загрузить через git локально, см. ниже).
   - Либо через терминал:
     ```
     git init
     git add .
     git commit -m "almaPLUS видеогид"
     git branch -M main
     git remote add origin https://github.com/<твой-аккаунт>/almaplus-guide.git
     git push -u origin main
     ```
3. В репозитории: **Settings → Pages → Source** → выбери ветку `main` и папку `/ (root)` → **Save**.
4. Через 1-2 минуты сайт будет доступен по адресу:
   ```
   https://<твой-аккаунт>.github.io/almaplus-guide/
   ```
5. Сгенерируй QR-код на эту ссылку (любой бесплатный QR-генератор) и распечатай/размести его для клиентов.

## Важно про размер видео

GitHub ограничивает:
- файл — до 100 МБ (через git push), до 25 МБ через веб-загрузку в браузере;
- репозиторий — мягкий лимит около 1 ГБ, рекомендуется не превышать 5 ГБ.

Если видео тяжелее — сжми их перед загрузкой (например через HandBrake, экспорт в H.264, 720p, битрейт ~1500-2500 kbps вполне достаточно для обучающего видео с экрана телефона). Это также ускорит загрузку видео у клиента на мобильном интернете.

## Проверка перед публикацией

- Открой `index.html` локально в браузере (просто двойной клик) — должен полностью работать выбор языка, список тем и плеер (кроме самого видео, если файлов ещё нет рядом).
- После загрузки на GitHub Pages — обязательно открой готовую ссылку с телефона и проверь, что видео реально проигрываются (мобильные браузеры иногда требовательны к кодеку — используй H.264/AAC, контейнер mp4).
